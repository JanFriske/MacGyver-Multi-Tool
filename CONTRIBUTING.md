# Contributing to MacGyver Multi-Tool

First off, thank you for considering contributing to MacGyver Multi-Tool! It's people like you that make open source such a great community.

## Where to start

There are many ways to contribute, from writing tutorials or blog posts, improving the documentation, submitting bug reports and feature requests or writing code which can be incorporated into MacGyver Multi-Tool itself.

## Bug Reports

- Ensure the bug was not already reported by searching on GitHub under [Issues](https://github.com/JanFriske/MacGyver-Multi-Tool/issues).
- If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/JanFriske/MacGyver-Multi-Tool/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

## Feature Requests

- We are always looking for new ideas to make MacGyver Multi-Tool better. If you have a feature in mind, please open an issue and describe what you would like to see.

## Pull Requests

- Fork the repo and create your branch from `main`.
- If you've added code that should be tested, add tests.
- If you've changed APIs, update the documentation.
- Ensure the test suite passes.
- Make sure your code lints.
- Issue that pull request!

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

We are looking forward to your contributions!
